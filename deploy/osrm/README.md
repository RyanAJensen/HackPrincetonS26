Place a prepared OSRM dataset in this directory before enabling the `routing` profile.

Expected files include:
- `region.osrm`
- `region.osrm.mldgr`
- companion files generated by `osrm-extract` / `osrm-partition` / `osrm-customize`

If this directory is empty, leave the `osrm` profile disabled and the backend will degrade gracefully when routing is unavailable.
